Ananta Stack Decisions

Every tool. What I picked. What I rejected and why. Optimised for: free or near-free at MVP, Indian-friendly (UPI, Kannada/Hindi, low bandwidth), open-source where possible, scales with revenue.

🇮🇳 India-first 💰 ₹0 to start 🔓 OSS-heavy 📈 Scales to 1L+ users 🎯 Solo-founder friendly

🧭 Guiding principles for every pick

  1. Free at MVP — Day-1 cost should be ₹0 for compute, storage, CDN, DB, monitoring. Paid only what you can't fake: domain (~₹800/yr) and Razorpay txn fees (2%).
  2. One language per layer — TypeScript everywhere except the AI worker (Python, because every Indic AI model is Python-first).
  3. Open-source preferred — no vendor lock-in. If a SaaS dies, you keep running.
  4. Indian models for Indian voices — AI4Bharat (Bengaluru, IIT Madras) handles Kanglish and Hindi better than any global TTS.
  5. Upgrade triggers tied to MRR — don't over-engineer for 1L users when you have 10. But pick tools that *can* scale there.
  6. Solo-founder rule — never pick a tool you can't operate alone. No Kubernetes-from-scratch, no Kafka cluster on day 1.

01 Frontend framework

The student app, parent dashboard, landing page. Must be fast on cheap Android phones with 3G, render Kannada/Devanagari cleanly, and feel joyful.

Next.js 14 (App Router) + React + TypeScript

Server-side rendering for SEO + fast first paint, client components for interactivity. App Router + React Server Components keep the JS bundle small — critical for ₹6k Android phones on 3G.

Chosen Free · MIT
Why this
  • Tailwind CSS already used in your existing mockups — no re-styling needed.
  • Vercel deploy is free (hobby tier) — perfect for staging. Production we'll self-host on Oracle to avoid Vercel's commercial limits.
  • Largest hiring pool in India for React devs when you eventually hire help.
  • Streaming SSR means the lesson page can start rendering before the AI worker finishes generating tomorrow's content.
Rejected
  • SvelteKit — smaller bundles, but tiny ecosystem in India + no LiveKit React-equivalent component library is as polished.
  • Remix — great patterns, but lost mindshare after the React Router merge; uncertain future.
  • Plain React + Vite — no SSR = slower first paint on cheap phones, worse SEO for landing page.
  • Flutter Web — bundle is 2MB+ before your code; brutal on 3G connections.

02 Backend API

The transactional brain — auth, payments, scheduling, attendance, quiz scoring, parent reports. Needs to be modular so you can extract services later.

NestJS (Node.js + TypeScript)

Opinionated framework with built-in DI, modules, guards, interceptors. Same language as the frontend → you share types via a single packages/db. Excellent for solo-dev productivity.

Chosen Free · MIT
Why this
  • Module-per-domain structure (auth, classes, payments, etc.) maps cleanly to your 4-plane architecture.
  • Built-in validation, swagger, queues, scheduling — fewer libraries to learn.
  • Prisma + NestJS is one of the most-documented combos online → fewer hours stuck.
  • Workers/scheduling via BullMQ comes built-in for the lesson-generation pipeline.
Rejected
  • Express — too unopinionated; you'll re-invent NestJS poorly as you grow.
  • FastAPI (Python) for everything — Python is great for AI but worse for high-RPS transactional APIs; harder to share types with the frontend.
  • Go + Fiber — fastest runtime, but doubles the languages you maintain solo.
  • Spring Boot — heavy, slow startup, JVM RAM appetite is brutal on free-tier VMs.
  • Django — couples to its ORM; doesn't fit a microservices-friendly path.

03 AI worker

Separate service that generates lesson content, calls TTS, renders animations, builds quizzes/notes/Memory Deck cards. Heavy GPU/CPU work that should NEVER block the main API.

FastAPI (Python 3.11) + Celery + Redis

Python because every Indic LLM/TTS model (AI4Bharat IndicTTS, IndicWhisper, Sarvam-1) is Python-only. FastAPI for sync HTTP, Celery for background jobs (lesson rendering can take 5–10 min).

Chosen Free · MIT/BSD
Why this
  • Best ML library ecosystem — PyTorch, Transformers, gTTS, Coqui, Piper, SadTalker all Python-native.
  • Async + types via Pydantic — closest experience to Nest/TS for the rest of your codebase.
  • Celery handles retry, scheduling, priority queues — exactly what overnight lesson generation needs.
Rejected
  • Node.js for AI — most Indic models simply don't have JS bindings.
  • Flask — older, no async, no built-in OpenAPI.
  • Ray Serve — great at scale but overkill for a single worker box; would consume your RAM on free tier.

04 Database

Stores users, lessons, attendance, quiz scores, payments, Memory Deck cards, pacing plan. Also needs vector storage for the AI to find related concepts during doubt-answering.

PostgreSQL 16 + pgvector + Prisma ORM

One database, two superpowers: relational for transactions, pgvector for semantic search (e.g. "find related quiz questions when student asks a doubt"). Prisma gives type-safe queries shared between frontend + API.

Chosen Free · PostgreSQL License
Why this
  • Supabase free tier hosts Postgres+pgvector free up to 500MB, 2GB egress/mo — easily fits 0→500 students.
  • pgvector means no separate vector DB (Pinecone $70/mo, Weaviate ops overhead) — one less thing to run.
  • Prisma migrations are reviewed in git → safer than raw SQL migrations.
  • Row-Level Security for multi-tenant student/parent isolation.
Rejected
  • MongoDB — bad fit for transactional money + heavy joins (parent ↔ child ↔ classes ↔ payments).
  • MySQL/MariaDB — no pgvector equivalent; weaker JSON support.
  • PlanetScale — used to be the obvious choice but killed free tier in 2024.
  • SQLite — fine for prototypes, won't survive concurrent live-class writes.
  • Pinecone / Weaviate / Qdrant as a separate vector DB — another piece to monitor.

05 Cache + queue

Session storage, BullMQ job queue, leaderboards, presence counts ("847 students watching live"), rate-limiting.

Redis 7 (self-hosted) + BullMQ

Industry default. Sub-millisecond reads. BullMQ on top gives you the job-queue features Celery offers, but for Node.

Chosen Free · BSD
Why this
  • Self-hosted on the Oracle VM for free; ~50MB RAM with the config we'll use.
  • Same Redis instance serves both BullMQ (Node side) and Celery (Python side) — one less service.
Rejected
  • Upstash Redis — generous free tier (10k commands/day) but billing is per-command; gets expensive fast at scale.
  • Memcached — no persistence, no streams, no pub/sub at the depth Redis offers.
  • RabbitMQ as queue — more ops overhead; BullMQ on Redis is plenty until you're past 100k jobs/day.
  • SQS — cloud lock-in, not free.

06 Auth + OTP

Students log in by phone number (most Karnataka families have one shared phone). Parents have separate accounts. Founder admin needs MFA.

Lucia Auth + bcrypt + MSG91 OTP

Self-hosted, framework-agnostic auth library. Sessions stored in Postgres. OTPs sent via MSG91 (Indian SMS gateway with ₹0.15/SMS pricing — cheapest in market).

Chosen Free · MIT (+SMS ₹0.15)
Why this
  • No vendor lock-in — Lucia is library code, not a service. Auth is too foundational to outsource.
  • MSG91 is DLT-registered (mandatory in India since 2021) and Indian companies trust it.
  • WhatsApp OTP option later via Meta Cloud API for power users (saves SMS cost).
Rejected
  • Auth0 — free up to 7,500 MAU then jumps to $240/mo. Hostile to scaling.
  • Firebase Auth — Google lock-in; phone-auth SMS is billed at $0.05/SMS = 8x MSG91.
  • Clerk — beautiful UX, but $25/mo + per-MAU fees; doesn't support Kannada UI strings well.
  • Supabase Auth — fine, but you're locked to their Postgres if you go this route.
  • NextAuth/Auth.js — good, but Lucia is leaner and more transparent.
  • Twilio Verify — $0.05/OTP and routes via foreign carriers (slow delivery in India).

07 LLM (lesson content generation)

Generates lesson scripts, quiz questions, notes summaries, Memory Deck flashcards, and answers student doubts in real time during live class.

Groq (Llama 3.3 70B + Llama 3.1 8B) + Sarvam-1 (Indic)

Groq serves Llama-family models at 500+ tokens/sec for FREE on their dev tier (30 req/min, 14k req/day). Sarvam-1 is an Indian-built 2B-param model trained specifically on Indic languages — best Kanglish output.

Chosen Free at MVP
Why this
  • Groq's speed (500 tok/s) means real-time doubt answering during live class — students don't wait.
  • Free dev tier covers 14,000 requests/day — enough for 10 lessons × 6 subjects × 2 mediums × ~50 LLM calls = ~6k/day.
  • Llama 3.3 70B matches GPT-4o on most reasoning tasks at 0 cost.
  • Sarvam-1 handles Kanglish code-mix far better than English-trained models (which often "correct" Kanglish into formal English or Kannada).
  • OpenRouter fallback when Groq rate-limits: free DeepSeek, free Gemini 2.0 Flash.
Rejected
  • OpenAI GPT-4o — $5/M input, $15/M output. At 6k requests/day × ~3k tokens each = ~$1,000/month from day 1. Murders the ₹0 promise.
  • Anthropic Claude — even pricier than OpenAI; no free production tier.
  • Self-hosted Llama on Oracle ARM — free VM has 24GB RAM but no GPU; ARM CPU inference of a 70B model takes 60+ sec/token. Unusable.
  • Cohere — generous free tier but weak on Indic languages.
  • Pure Sarvam for everything — small model struggles with multi-step reasoning needed for lesson scripts. Use it for Kanglish polish, not core reasoning.

08 TTS — English

For Anika (English teacher) and English-medium delivery of Maths, Science, Social Science.

Piper TTS (Rhasspy/OSS)

Tiny ONNX-based TTS that runs on CPU. ~50ms latency per sentence on the Oracle ARM VM. Several Indian-English voices already trained.

Chosen Free · MIT
Why this
  • Runs on the Oracle ARM CPU — no GPU needed.
  • Voices are downloadable (~30MB each) → you own them forever, no per-character billing.
  • Indian-English accent voices match Karnataka students' expectations better than American voices.
Rejected
  • ElevenLabs — gorgeous quality but $5/mo for only 10k chars; 60-min lesson script ≈ 30k chars → $80+/mo per teacher.
  • OpenAI TTS — $15/M characters; cheaper than ElevenLabs but still meaningful at scale.
  • Google Cloud TTS — $4/M chars but Indian accents are robotic.
  • Coqui XTTS v2 — better quality than Piper, but needs GPU. Plan B once we have an Oracle GPU instance.
  • Amazon Polly — AWS lock-in, similar pricing to Google.

09 TTS — Kanglish, Kannada, Hindi

For Vihaan, Praketa, Adhvara (Kanglish), Harini (Kannada), Amita (Hindi). No global TTS handles Kanglish properly — this is where we need an Indian model.

AI4Bharat IndicTTS (Indic-Parler-TTS)

Open-source TTS from AI4Bharat (IIT Madras + Bengaluru). Supports 22 Indic languages including Kannada and Hindi, with code-mixing support for Kanglish/Hinglish.

Chosen Free · MIT
Why this
  • Only Indian model trained on real Kanglish/Hinglish code-switched corpora.
  • Free weights via Hugging Face → run on your own infra.
  • Sarvam AI (commercial fork of AI4Bharat by the same team) offers a paid API for higher quality when revenue allows — graceful upgrade path.
Rejected
  • Google Cloud TTS (Kannada) — Kannada is supported but accent is wooden; cannot handle Kanglish code-mixing — it will either pronounce English words with heavy Kannada accent or vice versa.
  • ElevenLabs Multilingual v2 — supports Hindi but no Kannada, no Kanglish.
  • Azure Speech — has Kannada but same code-mix problem as Google.
  • OpenAI TTS — no Kannada at all.
  • Bhashini (Govt of India) — TTS quality is sub-Piper-level; integration paperwork is heavy.

10 AI avatar / lip-sync

Animates the anime teacher face to lip-sync the TTS audio. Critical for "AI teacher who actually teaches" feel.

Live2D Cubism (free tier) + Rhubarb Lip-Sync

2D rigged anime characters (much cheaper than 3D + more on-brand for the anime-face look you want). Rhubarb analyses the TTS audio and outputs viseme timings → drive the mouth shapes in Live2D.

Chosen Free for indie use
Why this
  • Live2D Free Edition is free for projects under $10M annual gross — you'll outgrow it pleasantly, not painfully.
  • Rhubarb is a tiny CLI that adds ~100ms to lesson rendering — no GPU needed.
  • Web-friendly — Live2D models render via Cubism Web SDK directly in browser → student doesn't download a 200MB video.
  • One model = infinite lessons — once you've rigged Vihaan once, every Vihaan lesson reuses the same file.
Rejected
  • SadTalker / Wav2Lip — full neural lip-sync; requires GPU per lesson render (~5 min on free GPU). Beautiful but blocks free-tier deployment.
  • HeyGen / D-ID / Synthesia — $30–$200/mo SaaS, generic Western avatars, lip-syncs poorly to Kannada/Kanglish.
  • Ready Player Me 3D — Western art style, doesn't match the anime brief.
  • Unreal MetaHuman — gorgeous but renders need a beefy GPU and giant downloads.
  • VRoid Studio — anime 3D models, but 3D in browser is heavy on cheap phones.

11 Animation engine (lesson scenes)

For the "animated scene" part of each lesson — e.g. burning magnesium for chemical reactions, balanced beams for algebra, India map for Social.

Manim Community Edition + Remotion (React video)

Manim (the 3Blue1Brown engine) renders mathematical/scientific animations from Python scripts. Remotion renders React components into MP4s — perfect for slide-style animations.

Chosen Free · MIT
Why this
  • Manim covers Maths/Science beautifully; the AI worker generates a Manim script per lesson, then renders to MP4.
  • Remotion covers Social/Language slides — reuse your existing Tailwind components.
  • Both render on CPU — Manim is FFmpeg under the hood; no GPU needed.
Rejected
  • After Effects + scripts — Adobe lock-in, expensive, can't run headless on free Linux.
  • Synthesia / DeepBrain — packages avatar + animation but SaaS pricing.
  • Stable Video Diffusion — beautiful but needs GPU, ~5min per 4-second clip.
  • Hand-drawn each lesson — doesn't scale beyond ~10 lessons solo.

12 Live class streaming

7–8 PM daily, AI teacher + recorded animation + live audience chat/polls/doubts. Needs sub-second latency for the "live" feel even though the teacher video is pre-rendered.

LiveKit (self-hosted) + HLS for late-joiners

WebRTC SFU for the live class itself (chat, polls, audio doubts). HLS stream from the same LiveKit room for browsers that want lower bandwidth or late-joiners on bad 3G.

Chosen Free · Apache 2.0
Why this
  • WebRTC sub-second latency for the interactive layer (raise hand, poll vote, type doubt).
  • Auto-generated HLS from LiveKit Egress for students on flaky connections — they trade 5–10 sec latency for stability.
  • Self-host free on the Oracle VM — handles ~500 concurrent until we need to upgrade.
  • LiveKit Cloud is the paid escape valve when revenue arrives.
Rejected
  • Agora — generous free tier (10k min/mo) then $1–$4/1000 min; expensive at scale, vendor lock-in.
  • 100ms — Indian SaaS, free tier 10k min/mo but capped. Self-hosting LiveKit gives more headroom.
  • Daily.co — great DX, but free tier is tiny.
  • Pure YouTube Live — no interactivity, no doubt-asking layer.
  • Zoom Webinar — ₹14k/mo per host, not how a tech-first brand looks.
  • Twilio Video — DEPRECATED in 2024.

13 Recording + on-demand (VOD)

Every class recorded and available on-demand. Students who missed the 7pm slot can catch up.

LiveKit Egress → MP4 → Cloudflare R2 + Cloudflare Stream

LiveKit records to MP4. Stored in R2 (S3-compatible, no egress fees). For playback, push to Cloudflare Stream once we cross 100 paying users (₹0.005/min played).

Chosen Free at MVP · Cheap at scale
Why this
  • R2 has zero egress fees — unlike S3 which charges $0.09/GB out. Critical: each recorded class is ~500MB, 1L students watching = 50TB egress = $4,500 on S3, ₹0 on R2.
  • Free tier: 10GB storage, 1M Class-A ops/mo — fits MVP.
  • Cloudflare Stream handles adaptive bitrate (3G→4G→WiFi) automatically.
Rejected
  • S3 / Backblaze + CloudFront — egress kills you at scale. R2 is purpose-built to fix that.
  • BunnyCDN — cheap ($0.005/GB) but not free; R2 wins on day 1.
  • Mux / Cloudflare Stream from day 1 — billed per-minute-stored AND per-minute-streamed; overkill before product-market fit.
  • YouTube unlisted — bandwidth is free but you give up control, ads risk, students can find it.

14 Object storage + CDN

Notes PDFs, materials, lesson assets, profile photos, Memory Deck images.

Cloudflare R2 + Cloudflare CDN

Same provider as live recording. S3-compatible API → Prisma + Nest can use the standard @aws-sdk/client-s3. Cloudflare CDN sits in front for free.

Chosen Free up to 10GB
Why this
  • Zero egress fees — same reason as VOD.
  • Cloudflare CDN is free — 270+ POPs globally including Bengaluru, Chennai, Mumbai.
  • Workers + R2 combo lets you do image resizing at the edge → smaller images for cheap phones.
Rejected
  • AWS S3 + CloudFront — egress fees + CloudFront has ~25 Indian POPs vs Cloudflare's 5+.
  • Backblaze B2 + Bunny — cheap, but two vendors instead of one.
  • Self-host MinIO on Oracle — uses your scarce VM disk; egress through Oracle is metered.

15 Compute hosting

Where the API, AI worker, LiveKit, Postgres, Redis actually run.

Oracle Cloud Always-Free ARM Ampere A1 (4 vCPU + 24 GB RAM)

The most generous free-forever VM in the industry. 4 ARM CPUs, 24GB RAM, 200GB block storage, 10TB/mo egress — all permanently free, no credit card expiry trick.

Chosen Free forever
Why this
  • Bigger than any other free tier — AWS gives 1 vCPU + 1GB for 12 months; Oracle gives 4 + 24 forever.
  • ARM = lower power = cheaper per cycle; modern Node/Python/Postgres all run native ARM.
  • Mumbai region available → 20ms latency to Karnataka students.
  • Comfortably runs API + Postgres + Redis + LiveKit + 1 AI worker in parallel.
Rejected
  • AWS Free Tier — 12 months only, then bills automatically. Tiny 1 vCPU.
  • GCP Free Tier — e2-micro is 0.25 vCPU and only in US regions.
  • Azure Free — 12 months only; B1S = 1 vCPU + 1GB.
  • Render / Railway / Fly.io — free tiers exist but apps sleep after inactivity; bad UX for a live-class platform.
  • Hetzner — €4/mo, not free, but a great upgrade target.
  • DigitalOcean — no free tier.
Caveat: Oracle's reputation for reclaiming idle free VMs is real but exaggerated; using the VM continuously (which we will) prevents reclamation. A weekly cron pings it to be safe.

16 Payments (UPI + cards)

₹299/mo subscription via UPI Auto-Pay. Must handle small-town parents who only have UPI, no card.

Razorpay (Subscriptions + UPI Auto-Pay)

Indian payment gateway with native UPI Auto-Pay (most parents won't enable card auto-debit but UPI auto-pay is trusted). Has the most coverage of Indian banks for failed-payment retries.

Chosen ~2% txn fee
Why this
  • UPI Auto-Pay is THE Indian recurring-payment rail — Razorpay supports it natively; most others don't.
  • No signup fee, no monthly fee — only 2% per transaction (1.99% for UPI, 2.36% for cards).
  • Subscriptions API handles billing cycles, retries, refunds — saves you 100s of hours.
  • Best-in-class GST invoicing — necessary for an Indian business.
Rejected
  • Stripe — entered India 2024 but UPI Auto-Pay support is still spotty; Indian banks know Razorpay better.
  • PayU — older, clunkier API, weaker docs.
  • Cashfree — strong but smaller bank coverage for failed-payment retries.
  • Paytm Payment Gateway — declining merchant adoption post the 2024 RBI issues.
  • Direct UPI Collect requests — works for one-off but you'd build the subscription logic yourself. Not worth it for 0.5% saved.

17 WhatsApp + SMS

Parent alerts for missed class, quiz scores, weekly test results, fee reminders. SMS only for OTP fallback.

Meta WhatsApp Business Cloud API + MSG91 (SMS fallback)

Meta Cloud API direct (no BSP middleman) → 1,000 free conversations/month from Meta. MSG91 for SMS-OTP when WhatsApp delivery fails.

Chosen Free 1k convos/mo
Why this
  • 1000 free conversations/mo from Meta = ~33/day = enough for first 100 parents.
  • Going direct to Meta skips the BSP markup (Gupshup, Karix charge 30–50% on top).
  • Utility category conversations (homework reminders, attendance) are ~₹0.115 each — dirt cheap.
  • WhatsApp template approval ~24 hours; we'll pre-approve all 8–10 templates we need.
Rejected
  • Gupshup / Karix / WATI / AiSensy — 30–50% markup; useful only if you want a no-code drag-drop builder, which you don't.
  • Twilio WhatsApp — global pricing, 2–3x Indian alternatives.
  • MSG91 WhatsApp — they have it, but markup vs going direct to Meta.

18 Email

Receipts, password reset, optional weekly parent digest.

Resend (free tier) → AWS SES at scale

Resend gives 3,000 emails/mo + 100/day free; gorgeous DX. AWS SES at $0.10/1000 emails is the long-term home.

Chosen Free
Why this
  • Resend is the developer-grade email service — modern API, React templates, free tier is enough until ~1k paying users.
  • SES upgrade path is industry-standard at scale.
Rejected
  • SendGrid — free tier slashed to 100/day in 2024; deliverability declining.
  • Mailgun — no free tier anymore.
  • Self-hosted Postfix — Karnataka ISPs WILL flag your IP. Don't do this.

19 PDF generation (notes downloads)

Auto-generated notes PDFs for download, parent reports, fee receipts.

Playwright headless Chromium + React templates

Render the same React component to HTML → screenshot to PDF. Looks identical to what students see on screen.

Chosen Free · Apache 2.0
Why this
  • One template, two outputs (browser view + PDF) — reuses your existing Tailwind components.
  • Handles Kannada/Devanagari fonts natively — unlike older PDF libs that need font registration gymnastics.
Rejected
  • react-pdf — separate component model; can't reuse your normal React.
  • wkhtmltopdf — abandoned; broken on modern Chromium.
  • Puppeteer — fine but Playwright has better Linux ARM support.
  • PDFKit — programmatic only; awful for designed documents.

20 Analytics

Who's actually using the product, where do students drop off, which lessons are loved.

PostHog Cloud (free tier) + Cloudflare Web Analytics

PostHog for product analytics (funnels, heatmaps, session replays) — 1M events/mo free. Cloudflare Web Analytics for marketing (privacy-respecting, ad-blocker-resistant).

Chosen Free
Why this
  • PostHog is open-source — self-host the day you cross 1M events/mo.
  • Session replay is gold for solo debugging: when a parent says "the quiz didn't work", you watch their session.
  • Cloudflare Web Analytics is ad-blocker-proof because it works at the CDN layer.
Rejected
  • Google Analytics 4 — useless for product analytics; ad-blockers nuke it; clunky UI.
  • Mixpanel — paid after 100k events/mo.
  • Amplitude — paid after 50k events/mo and aggressive on upsell.
  • Plausible Cloud — beautiful but no funnels/replays.

23 Container orchestration

Docker Compose (MVP) → k3s (when scaling)

Compose for the single Oracle VM. k3s (lightweight Kubernetes) when we need 2+ VMs across regions.

Chosen Free · Apache 2.0
Why this
  • Compose is YAML you can read in 10 minutes. Don't introduce K8s before you need it.
  • k3s is a 100MB single binary that gives you Kubernetes without the operational pain.
Rejected
  • Full Kubernetes from day 1 — solo-founder rule.
  • Nomad — fine but ecosystem is smaller; k3s is more "boring tech".
  • Docker Swarm — dying.

27 Search (lessons, materials, doubts)

Postgres full-text + pgvector

Both keyword search ("French Revolution") and semantic search ("ಫ್ರೆಂಚ್ ಕ್ರಾಂತಿ") in the same database, no extra service.

Chosen Free
Rejected

  • Meilisearch / Typesense — great, but another service. Postgres FTS is "good enough" until 50k+ documents.

Chosen Free
